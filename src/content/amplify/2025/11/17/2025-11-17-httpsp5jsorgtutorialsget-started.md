<h3>Default Code</h3>
<p>All p5.js projects include the p5.js library and three files: <a href="https://www.classes.cs.uchicago.edu/archive/2021/spring/11111-1/happycoding/p5js/web-dev.html#:~:text=The%20index.,the%20page%20using%20HTML%20tags!/"><code>index.html</code></a>, <a href="https://happycoding.io/tutorials/p5js/web-dev/">style.css</a>, and <code>sketch.js</code>. Make changes on the canvas by adding code to the sketch.js file. New p5.js projects begin with the following code in the sketch.js file:</p>
<p><code>

function draw() {

<p>Every sketch.js file begins with two main functions: <a href="https://p5js.org/reference/p5/setup/"><code>setup()</code></a> and <a href="https://p5js.org/reference/p5/draw/"><code>draw()</code></a>.</p>
<p><a href="https://p5js.org/reference/p5/function/">Functions</a> are sequences of instructions that perform specific tasks.</p>
<p>When the code in sketch.js is executed:</p>
<ul><li><a href="https://p5js.org/reference/p5/setup/"><code>setup()</code></a> is called and runs one time. The default value (220) sets the color of the canvas to gray.</li></ul></li></ul>
<p>Visit the <a href="https://p5js.org/reference/">p5.js reference</a> pages for <a href="https://p5js.org/reference/p5/setup/"><code>setup()</code></a>, <a href="https://p5js.org/reference/p5/draw/"><code>draw()</code></a>, <a href="https://p5js.org/reference/p5/createCanvas/"><code>createCanvas()</code></a>, and <a href="https://p5js.org/reference/p5/background/"><code>background()</code></a> to learn more.</p>

<h3>p5.js Canvas Coordinates</h3>
<p>The canvas comes with an invisible coordinate system that begins at a horizontal location of 0 and vertical location of 0 in the top-left corner.</p>
<figure><img alt="A diagram of a HTML &lt;canvas&gt; element shows a grid pattern overlay with the horizontal axis representing x-coordinates, the vertical axis representing y-coordinates, and each intersection of the grid representing specific (x,y) coordinate pairs. An arrow along the x-axis shows that x-coordinates increase as they move to right on the canvas. An arrow along the y-axis shows that y-coordinates increase as they move down on the canvas." src="https://p5js.org/_astro/coordinates.DW7JVAkD_Z13e1Vf.webp"/><figcaption>A diagram of a HTML &lt;canvas&gt; element shows a grid pattern overlay with the horizontal axis representing x-coordinates, the vertical axis representing y-coordinates, and each intersection of the grid representing specific (x,y) coordinate pairs. An arrow along the x-axis shows that x-coordinates increase as they move to right on the canvas. An arrow along the y-axis shows that y-coordinates increase as they move down on the canvas.</figcaption></figure>
<p>As an object on the canvas moves to the right of this point, its horizontal location increases. As an object on the canvas moves down from this point, its vertical location increases. As an object moves to the left on the canvas, its x-coordinate decreases; as an object moves up on the canvas, its y-coordinate decreases. The following examples display values for the x- or y-coordinates of a circle as its position changes on the canvas:</p>
<ul><li><a href="https://editor.p5js.org/Msqcoding/full/AM5ZwrmNo/">This example</a> displays the x-coordinate of a circle as it moves horizontally on the canvas.</li><li><a href="https://editor.p5js.org/Msqcoding/full/jZeTUjZfZ/">This example</a> displays the y-coordinate of a circle as it moves vertically on the canvas.</li></ul>
<p>The maximum values for the horizontal and vertical position of the canvas are set by the <a href="https://p5js.org/reference/p5/createCanvas/"><code>createCanvas()</code></a> function. The first number in <a href="https://p5js.org/reference/p5/createCanvas/"><code>createCanvas()</code></a> is the x-coordinate of the right edge, and the second number is the y-coordinate of the bottom edge.</p>
<p>Learn more about the HTML canvas coordinate system and shapes on <a href="https://p5js.org/reference/">this p5.js reference page</a>.</p>

<h3>Fill Order</h3>
<p>The default value for <a href="https://p5js.org/reference/p5/fill/"><code>fill()</code></a> is white. This means that if <a href="https://p5js.org/reference/p5/fill/"><code>fill()</code></a> never appears in <a href="https://p5js.org/reference/p5/draw/"><code>draw()</code></a>, all shapes will be white by default. To change the color of any shape on the canvas, you must call <a href="https://p5js.org/reference/p5/fill/"><code>fill()</code></a> before drawing the shape. The default value for <a href="https://p5js.org/reference/p5/stroke/"><code>stroke()</code></a> is black. To change the color and shape of the outline, <a href="https://p5js.org/reference/p5/stroke/"><code>stroke()</code></a> and <a href="https://p5js.org/reference/p5/strokeWeight/"><code>strokeWeight()</code></a> must be called before the shape is drawn. When drawing multiple shapes on the canvas, call <a href="https://p5js.org/reference/p5/fill/"><code>fill()</code></a>, <a href="https://p5js.org/reference/p5/stroke/"><code>stroke()</code></a>, and <a href="https://p5js.org/reference/p5/strokeWeight/"><code>strokeWeight()</code></a> each time the color of a shape or outline changes.</p>

<p>Since <a href="https://p5js.org/reference/p5/draw/"><code>draw()</code></a> executes the code within the brackets over and over again, shapes are drawn on the canvas like layers. Code that appears first will be drawn first, and code that appears further down in <a href="https://p5js.org/reference/p5/draw/"><code>draw()</code></a> will be drawn after. This is why we must set <a href="https://p5js.org/reference/p5/fill/"><code>fill()</code></a>, <a href="https://p5js.org/reference/p5/stroke/"><code>stroke()</code></a>, and <a href="https://p5js.org/reference/p5/textSize/"><code>textSize()</code></a> before we add shapes or text to the canvas. p5.js needs to know the color or size of shapes and text <em><strong>before</strong></em> they are drawn. This also allows for overlapping shapes and can sometimes hide shapes behind others! Be sure to check the x- and y-coordinates of your shapes to see if they are behind other shapes.</p>

<h3>Data Types</h3>
<p>Some functions in p5.js need <em>arguments</em> that are <em>string</em> data types, while other functions need arguments that are <em>number</em> data types.</p>
<ul><li><a href="https://p5js.org/reference/p5/String/">Strings</a>: represented by text and always surrounded by quotation marks (<code>""</code>)</li><li><a href="https://p5js.org/reference/p5/number/">Numbers</a>: represented by a number </li></ul>
<p>For example:</p>
<ul><li><a href="https://p5js.org/reference/p5/circle/"><code>circle()</code></a> uses 3 <a href="https://p5js.org/reference/p5/number/">numbers</a>: a number that represents its x-coordinate, a number that represents its y-coordinate, and a number that represents its size in pixels. </li><li><a href="https://p5js.org/reference/p5/text/"><code>text()</code></a> uses a <a href="https://p5js.org/reference/p5/String/">string</a> that represents the text displayed, and 2 <a href="https://p5js.org/reference/p5/number/">numbers</a>: a number that represents its x-coordinate and a number that represents its y-coordinate. </li><li><a href="https://p5js.org/reference/p5/fill/"><code>fill()</code></a>, <a href="https://p5js.org/reference/p5/stroke/"><code>stroke()</code></a>, and <a href="https://p5js.org/reference/p5/background/"><code>background()</code></a> use both <a href="https://p5js.org/reference/p5/number/">numbers</a> and <a href="https://p5js.org/reference/p5/String/">strings</a>:<ul><li>1 number for gray-scale color</li></ul></li></ul>
