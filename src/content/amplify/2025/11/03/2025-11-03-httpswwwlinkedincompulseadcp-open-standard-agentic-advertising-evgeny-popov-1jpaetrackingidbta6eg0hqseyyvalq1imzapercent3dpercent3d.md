<h3> Evgeny Popov </h3>
<p>Connect once. Operate anywhere.</p>
<p>Autonomous agents are no longer a lab toy in ad tech. They already run parts of live campaigns. The blocker is fragmentation: every platform speaks its own language. AdCP fixes this with an open, extensible standard for how intelligent agents talk to advertising platforms. It unifies three core jobs - Audience Activation, Curation, and Media Buy- so agents can plan and execute without custom builds.</p>
<p>Built on Anthropic’s Model Context Protocol (MCP). MCP handles auth, permissions, and context. AdCP uses that foundation so agents and platforms can exchange intent, options, and results in a clean, secure way. No rip-and-replace. Adopt it once; operate everywhere.</p>
<h3>What AdCP enables</h3>
<ul> <li>One interface, many platforms. Connect once across your stack.</li> <li>Consistent intent. Express audiences, budgets, and outcomes the same way everywhere.</li> <li>Less glue code. Replace a patchwork of proprietary APIs with one spec.</li> <li>Uniform data. Inventory, pricing, performance, and delivery in a standard shape.</li> </ul>
<p>Audience Activation, Curation, and Media Buy are live today in the open-source reference implementation.</p>
<h3>Where agents win first (Ari’s lens)</h3>
<p> <a data-test-link="" data-tracking-control-name="article-ssr-frontend-pulse_little-mention" data-tracking-will-navigate="" href="https://www.linkedin.com/in/aripaparo?trk=article-ssr-frontend-pulse_little-mention"> Ari Paparo </a>  take is practical and helpful: agents shine when they remove busywork and reduce error.</p>
<ul> <li>Single-platform automation. If thousands of users type the same steps into your UI, agents will do it faster and better.</li> <li>Cross-platform with the same customer. Think creative + DSP, or data broker + DSP. Agents can pass precise requirements and payloads between parties and cut human rework.</li> </ul>
<p>Ari’s favorite early win: creative/asset handling. Clear schemas stop “AI guesswork.” Instead of “upload a banner,” the agent knows the exact asset type, size, format, and file limits. Fewer back-and-forths. Faster time to live.</p>
<p>Signals discovery is also promising: let agents query segment catalogs across providers, apply filters, and return a shortlist that fits the brief. Caveat (again from Ari): incentives and trust matter. Data sellers will market their segments; buyers need validation and governance. AdCP can standardize “what” and “how,” but the market still needs checks on “how good.”</p>

<h3>Media buying: be ambitious, but grounded</h3>
<p>Ari is bullish on automation, cautious on agent-led transactions at scale. History shows tough issues in “programmatic direct”: rate cards, price discovery, and uneven data. Agents don’t erase those tensions. Two pragmatic paths forward:</p>
<ol> <li>Intermediary-led execution. SSPs and cross-publisher networks can package supply and assume delivery risk. Think “curation by wire” instead of static deal IDs.</li> <li>Data-driven PG/PMP at scale. A buying agent with a tight audience definition can pre-negotiate across many publishers where the signal is common in an ad server or curation layer.</li> </ol>
<p>The takeaway: Media Buy via agents is viable, but the value likely accrues first to large publishers and intermediaries who can aggregate supply, signals, and accountability.</p>
<h3>How it works (mirrors how teams think)</h3>
<ol> <li>Intent. The agent declares objectives and constraints.</li> <li>Discovery. Platforms respond with structured options for audiences, inventory, and pricing.</li> <li>Activation. Approved plans launch using the Media Buy workflow.</li> <li>Measurement. Results return in a consistent format for analysis and reporting.</li> </ol>
<p>All exchanges are secure and context-aware per MCP.</p>
<h3>Who uses AdCP</h3>
<ul> <li>Advertisers &amp; agencies. Access audiences and inventory directly. Plan on intent and outcomes, not line-item busywork. Get one view of pricing and performance across channels.</li> <li>Ad tech platforms. Implement once to make exchanges, ad servers, and measurement tools agent-ready—while keeping your product edge.</li> <li>Publishers &amp; media owners. Make packages and pricing discoverable to agent buyers. Offer premium sponsorships and niche segments with less lift. Transact with transparency and control.</li> </ul>

<h3>Open, neutral, extensible</h3>
<p>AdCP is open source and industry-led. A neutral consortium governs the standard with public docs, working-group review, and versioned releases. The core schema is extensible, with optional fields for proprietary data. Licensed under MIT to be easy to implement and hard to monopolize.</p>

<h3>Adoption path (start small, scale fast)</h3>
<ol> <li>Read the spec - GitHub</li> <li>Integrate one workflow (Audience Activation, Curation, or Media Buy) that maps to your product.</li> <li>Validate with conformance tests.</li> <li>Join the working group to guide releases, incentives, and guardrails.</li> </ol>
<h3>Bottom line</h3>
<ul> <li>Agents reduce waste and speed execution.</li> <li>Clear schemas beat AI “best guesses.”</li> <li>Signals and creative win now; media buying scales with intermediaries and strong governance.</li> <li>Standards don’t fix incentives, but they make progress possible.</li> </ul>
