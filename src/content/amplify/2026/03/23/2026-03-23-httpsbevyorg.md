<h2>Data Driven</h2>
<p>All engine and game logic uses Bevy ECS, a custom Entity Component System</p>
<ul><li><b>Fast:</b> Massively Parallel and Cache-Friendly</li><li><b>Simple:</b> Components are Rust structs, Systems are Rust functions</li><li><b>Capable:</b> Queries, Global Resources, Local Resources, Change Detection, Lock-Free Parallel Scheduler</li></ul>
<figure><img alt="ECS code" src="https://bevy.org/assets/ecs.svg"/><figcaption>ECS code</figcaption></figure>
<figure><img alt="Pixel art sprite of a person" src="https://bevy.org/assets/sprite.png"/><figcaption>Pixel art sprite of a person</figcaption></figure>
<h2>2D Renderer</h2>
<p>Render real-time 2D graphics for games and apps</p>
<ul><li><b>Features:</b> sprite sheets, dynamic texture atlases, cameras, textures, and materials</li><li><b>Extensible:</b> custom shaders, materials, and render pipelines</li><li><b>Common Core:</b> builds on top of Bevy's Render Graph</li></ul>
<h2>3D Renderer</h2>
<p>A modern and flexible 3D renderer</p>
<ul><li><b>Features:</b> lights, shadows, cameras, meshes, textures, materials, gltf loading</li><li><b>Extensible:</b> custom shaders, materials, and render pipelines</li><li><b>Common Core:</b> built on top of Bevy's Render Graph</li></ul>
<figure><img alt="3D boat model" src="https://bevy.org/assets/boat.png"/><figcaption>3D boat model</figcaption></figure>
<figure><img alt="Render graph" src="https://bevy.org/assets/render_graph.svg"/><figcaption>Render graph</figcaption></figure>
<h2>Render Graph</h2>
<p>Compose custom render pipelines using a graph structure</p>
<ul><li><b>Parallel:</b> Render Graphs are automatically rendered in parallel</li><li><b>Modular:</b> Build composable and reusable render logic using Render Graph nodes</li><li><b>Backend Agnostic:</b> Not tied to a specific graphics API</li></ul>
<h2>Animation</h2>
<p>A capable animation system</p>
<ul><li>Skeletal rig animation driven by an ECS-based joint API</li><li>Play multiple animations at the same time by smoothly blending between them</li><li>Use blend shapes / morph targets to animate vertices directly</li><li>Import animations from GLTF files</li></ul>
<figure><img alt="An animated poly style model of a fox running" src="https://bevy.org/assets/fox.webp"/><figcaption>An animated poly style model of a fox running</figcaption></figure>
<figure><img alt="Microsoft, Apple, and Linux logos" src="https://bevy.org/assets/platform-icons.svg"/><figcaption>Microsoft, Apple, and Linux logos</figcaption></figure>
<h2>Bevy UI</h2>
<p>A custom ECS-driven UI framework built specifically for Bevy</p>
<ul><li>Built directly on top of Bevy's ECS, Renderer, and Scene plugins</li><li>Compose UIs dynamically in code or declaratively using the Bevy Scene format</li><li>Use a familiar "flex box" model to layout your UIs</li></ul>
<figure><img alt="UI dialog window diagram" src="https://bevy.org/assets/bevy_ui.svg"/><figcaption>UI dialog window diagram</figcaption></figure>
<figure><img alt="2D scene of a square upon grass under the sun" src="https://bevy.org/assets/scene.svg"/><figcaption>2D scene of a square upon grass under the sun</figcaption></figure>
<h2>Scenes</h2>
<p>Create, save, and load ECS worlds using Bevy's Scene system</p>
<ul><li><b>Loading:</b> Loading scenes preserves entity IDs (useful for save games)</li><li><b>Instancing:</b> Instancing creates linked duplicates of scenes with new entity IDs</li><li><b>Hot Reloading:</b> Changes to scene files are automatically applied to running apps</li></ul>
<h2>Sound</h2>
<p>Load audio files and play them on demand</p>
<ul><li>Load audio files as Assets</li><li>Play audio Assets using Audio entities</li></ul>
<figure><img alt="Music notes" src="https://bevy.org/assets/sound.svg"/><figcaption>Music notes</figcaption></figure>
<figure><img alt="Hot reloading diagram" src="https://bevy.org/assets/hot_reloading.svg"/><figcaption>Hot reloading diagram</figcaption></figure>
<h2>Hot Reloading</h2>
<p>Get instant feedback on your changes without app restarts or recompiles</p>
<ul><li>Asset changes are immediately reflected in running Bevy apps</li><li>You can currently hot-reload scenes, textures, and meshes</li><li>Any asset type can be integrated</li></ul>
<h2>Productive Compile Times</h2>
<p>Game development is an iterative process. You can't afford to wait for compiles</p>
<ul><li>With Bevy you can expect 0.8-3.0 seconds with the "fast compiles" configuration</li><li>Compare that to other popular Rust game engines, which can take over 30 seconds to compile a single newline insertion!</li></ul>
