<h2>HTML Minifier Next 5, a Quick Upgrade Guide</h2>
<p>Published on Feb 2, 2026, filed under <a href="https://meiert.com/topics/development/">development</a>, <a href="https://meiert.com/topics/html/">html</a>, <a href="https://meiert.com/topics/optimization/">optimization</a>. ( <a href="https://shareopenly.org/share/?url=https://meiert.com/blog/html-minifier-next-5/&amp;text=HTML%20Minifier%20Next%205%2C%20a%20Quick%20Upgrade%20Guide">Share this post</a>, e.g. <a href="https://s2f.kytta.dev/?text=HTML%20Minifier%20Next%205%2C%20a%20Quick%20Upgrade%20Guide,%20by%20%40j9t%40mas.to%3A%0A%0Ahttps://meiert.com/blog/html-minifier-next-5/">on Mastodon</a> or <a href="https://bsky.app/intent/compose?text=HTML%20Minifier%20Next%205%2C%20a%20Quick%20Upgrade%20Guide,%20by%20%40meiert.com%3A%0A%0Ahttps://meiert.com/blog/html-minifier-next-5/">on Bluesky</a>.)</p>
<p>The next major release of <a href="https://www.npmjs.com/package/html-minifier-next/v/5.0.0">HTML Minifier Next (HMN)</a> just shipped, and contrary to previous major releases, this one comes with <a href="https://github.com/j9t/html-minifier-next/blob/main/CHANGELOG.md">quite a few</a> breaking DX and UX improvements.</p>
<p>To make it easier to work through the breaking changes, here’s a quick checklist:</p>
<ul>
<li>
<p>File extensions: <strong>If you do <em>not</em> limit the file types passed to HMN and you use HMN on files other than .html, .htm, .xhtml, or .shtml</strong>, you need to either enable the specific file types you use, or enable all file types (<code>fileExt: "*"</code> or <code>--file-ext='*'</code>).</p>
</li>
<li>
<p>“HTML 5” mode: <strong>If you explicitly used the <code>html5</code> option</strong> (<code>html5</code> or <code>--no-html5</code>), you should remove it—this mode is always on now and cannot be disabled.</p>
</li>
<li>
<p>Sorting by class names: <strong>If you sort by class names</strong>, you need to rename the parameter to use the plural (<code>sortClassNames</code> or <code>--sort-class-names</code>).</p>
</li>
<li>
<p>URL minification configuration: <strong>If you use a custom configuration for <code>minifyURLs</code></strong>, you need to remove unsupported properties (only <code>site</code> is accepted with the updated configuration).</p>
</li>
</ul>
<p>Depending on your preferences:</p>
<ul>
<li>
<p>Presets: <strong>If you care about the output of presets</strong>, check the output and explicitly enable missing or disable unwanted settings (the presets were optimized for an improved—though opinionated—starter and expert experience).</p>
</li>
<li>
<p>Auto-generated tags: <strong>If you rely on certain tags that were <em>inserted</em> in the past</strong>, you need to enable <code>includeAutoGeneratedTags</code> (<code>includeAutoGeneratedTags: true</code> or <code>--include-auto-generated-tags</code>).</p>
</li>
</ul>
<p>For additional details, please check <a href="https://github.com/j9t/html-minifier-next/blob/main/CHANGELOG.md">the changelog</a>. If you run into any problems, please <a href="https://github.com/j9t/html-minifier-next/issues">file an issue</a>.</p>
<p>The update contains a few more improvements around security, performance, and developer experience, and should form an even better foundation to extend HMN further.</p>
<p>Stay tuned—and if you like the project as a powerful HTML optimization tool, spread the word and consider <a href="https://github.com/j9t/html-minifier-next?sponsor=1">supporting the work on it</a>! Thanks!</p>
