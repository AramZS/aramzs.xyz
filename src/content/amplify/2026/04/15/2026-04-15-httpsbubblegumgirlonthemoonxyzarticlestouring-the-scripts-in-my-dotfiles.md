<p>hey friends! this is late, but seeing as this is my first post of the year, happy 2026!</p>
<p>i love to write random shell scripts, either because they automate something i do all the time, or simply because i'm bored. a lot of my recent scripts have been lifted/inspired by the ones found in Evan Hahn's dotfiles, which he wrote about in the post <a href="https://evanhahn.com/scripts-i-wrote-that-i-use-all-the-time/"><i>Scripts I wrote that I use all the time</i></a>. as this post has greatly inspired my own dotfiles and how i automate my personal tasks, i decided i'd just straight up rip off that post and do a similar one for my own scripts! sorry Mr. Hahn— i hope this is okay!</p>
<p>i'll preface this with a few things: first off, my scripts are <em>messy</em>. i'm still somewhat a beginner to scripting, and i'm very much not a programmer; if the script works even a little bit, it goes in my dotfiles repo and i only touch it if there's something wrong. second, i rely on lots of external dependencies often; these are to automate personal tasks, most of which involve making it easier to use an existing tool, like <code>imagemagick</code>, <code>ffmpeg</code>, <code>yt-dlp</code>, and more.</p>
<p>one of these dependencies is <code>gum</code> by Charm CLI, which, in their words, allows you to make "glamorous shell scripts," AKA, make your shell scripts <em>interactive</em>. though i realize i can replace lots of this interactivity with tools that have been around longer, such as <code>read</code> for text input, i just prefer the way <code>gum</code> does it! it's not only very cute, but more importantly, it is <em>intuitive</em> and easy to get started with. ease of use matters a lot to me! i recommend you give <a href="https://github.com/charmbracelet/gum"><code>gum</code></a> a shot, it really makes shell scripting fun. (obligatory in this day and age: i am in no way sponsored to say all of that, i'm just some guy who really likes what Charm does.)</p>
<p>some scripts have a "tools" suffix; this means that they collect various code snippets with a common theme, and presents them in a menu that you can choose from. each option usually runs as a bash loop, as they are intended to run in a directory applying to all the files contained.</p>
<p>lastly, i won't be covering all of my scripts because some of them suck ass. also i'm linking to the <a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles">tangled mirror of my dotfiles</a>, in case my personal forge ever goes down</p>
<p>okay, onto the scripts!</p>
<h3>general file management</h3>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/eb07519aca57cb1016aae51bcf1d5fd8cef1b555/config/.local/bin/copy"><code>copy</code></a> will, well, copy anything it's told to. i use it most often with a pipe; for example, i'll do <code>echo "thing to be copied" | copy</code>, and then if i check my clipboard or paste it, it'll result in that copied string.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/eb07519aca57cb1016aae51bcf1d5fd8cef1b555/config/.local/bin/paste"><code>paste</code></a> is basically the inverse of the above; it just pastes stuff! i can use it similarly to copy, like if i want to put the clipboard contents in a file: <code>paste &gt; my_epic_file.txt</code>.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/6d7f08d3c829377d45657f768b79c9d954c76e3d/config/.local/bin/extlan"><code>extlan</code></a> is short for <i>extract language</i>; this means that it'll extract ZIP archives with certain language encodings, specifically CJK, without resulting in mangled <a href="https://en.wikipedia.org/wiki/Mojibake?useskin=vector">mojibake</a> files. it prompts for the type of encoding (at the moment i've only needed this for japanese &amp; korean encodings), and when one is chosen, it prompts for the name of the file to be extracted.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/15ee5531abb754de30b8894fc1aaf0094250adae/config/.local/bin/md2html"><code>md2html</code></a> is a mess but it's my mess okay. i love it. this script will convert markdown to HTML, using a variety of cluttered, probably sometimes useless dependencies. it has two options: paste in your markdown, or pull it from a file. i use the paste option the most, but both use very similar code. most of the heavy-lifting (such as accepting text input, or fuzzy-searching through a file list) is done by love of my life <code>gum</code>, but the text conversion itself is handled by <code>pandoc</code>, and the resulting HTML entities of characters like double quotes and ampersands are handled by a perl one-liner. i use this script <em>all the fucking time</em>, mainly to process my fanfiction as i prepare it to be posted, copying from the obsidian editor right into the script. i love it so much.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/73ae21fc232038a2ac6de58348cbf097e41c38be/config/.local/bin/txttools"><code>txttools</code></a> gathers several text processing functions into one script: <code>unwraptxt</code> will accept pasted-in wrapped text and unwrap it using a perl one-liner; <code>wraptxt</code> does the same but for wrapping text by a certain amount of characters; and lastly, <code>wdcount</code> will tell you the word count of pasted-in text. i recognize that a lot of these things can be done with simple tools like <code>awk</code>, <code>wc</code>, and others, but i am a lazy fuck and love to paste in text with <code>gum write</code>.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/11462a65786de436d0fe0c1ecc4acd0272854959/config/.local/bin/mksh"><code>mksh</code></a> allows me to quickly make scripts and either place them in my <code>$PATH</code> (specifically <code>~/.local/bin</code> with the rest of my scripts), or in the current working directory. it adds a shebang and makes the script executable before i open it.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/98b9686b08968a9059c708dd414b523d1e27cd49/config/.local/bin/stripexif"><code>stripexif</code></a> uses <a href="https://exiftool.org/"><code>exiftool</code></a> to strip the EXIF data (minus orientation/rotation) of any image supplied to it. good for privacy nerd stuff.</p>
<h3>life utilities</h3>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/45eaf63c57ca086224d1e523c6e6c891bfe27ecf/config/.local/bin/mini_timer"><code>mini_timer</code></a> is well, a mini timer! it's "mini" because it only works in minutes. it uses <a href="https://github.com/caarlos0/timer"><code>timer</code></a> by caarlos0 to do the heavy lifting. when the timer is done, it will send a system notification, as well as playing a little chime sound, and even a text-to-speech line informing me that it's done, in case i miss the notification (and i also almost always have my headphones on).</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/789bde0582a2f1b0da3f48cad496a2e22325ac0c/config/.local/bin/mvpn"><code>mvpn</code></a> is a wrapper around my VPN (<a href="https://mullvad.net/">mullvad</a>) CLI, and allows me to quickly launch any application in my <code>$PATH</code> outside of my VPN's connection (split-tunneled).</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/7c664b21611c649e8fdd16b9db3e411c3a19a39a/config/.local/bin/mvpnall"><code>mvpnall</code></a> (also linked to <code>splitvpn</code>) is a semi-complex script that achieves a simple goal: split-tunneling a bunch of my commonly used applications away from my VPN. sometimes i'll have my VPN on for some background tasks, and then having it on will affect my usual daily applications like my browser, IRC client, discord, and more. running this once will auto-magically split-tunnel all of those applications out from the VPN, and if the VPN isn't currently connected, it will do that as well.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/b52f34cb1554901a67959ef461dc889c38bdcea0/config/.local/bin/pick"><code>pick</code></a> uses <code>xcolor</code> to let me color pick anything on screen. it then copies the hex code to my clipboard, and sends a notification upon completion. i have this tied to a hotkey for ease of use!</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/67da6a81c4ead6362a3cc8cc364131d5ca7cb394/config/.local/bin/pomodoro"><code>pomodoro</code></a> is basically a souped-up version of <a href="https://gist.github.com/bashbunni/3880e4194e3f800c4c494de286ebc1d7">bashbunni's <code>.zshrc</code> pomodoro timers</a>; in other words, it's over-engineered for something i barely use lol. the various options are for different lengths of time: a quick run, which is 15 minutes, a break which lasts for 20 minutes, a work run that goes for 45 minutes, and lastly, a custom option that allows me to enter any duration i want.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/b09372a64089409382873ac62da667e6964f94ef/config/.local/bin/screenshot"><code>screenshot</code></a> &amp; <a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/b09372a64089409382873ac62da667e6964f94ef/config/.local/bin/screenshotpick"><code>screenshotpick</code></a> have completely replaced my distro's built-in GUI screenshot utilities. basically, on both my PC and laptop each, i have one big screenshot folder, and in my PC's case, i've been maintaining it since 2020. each file is numbered sequentially. before these scripts, i used to take the screenshot with a hotkey, which would launch the GUI; then i'd have to <em>manually</em> save the file, after my massive folder of screenshots has loaded, which can take a few seconds, so i could see what the last number was. this was miserable!!! i also hated the GUI's click-and-drag function for screenshots. so after taking inspiration from a github script, i ended up with these two, which are both bound to two different hotkeys. they're efficient as hell and perfect for my highly opinionated approach to screenshot taking and saving!</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/6fab3a4f08fe172186cfa10240dbcc59a119410b/config/.local/bin/share"><code>share</code></a> quickly beams any file to a file server of mine that holds a bunch of randomly dumped files that i need a link to. it uses <code>rsync</code> to send the files over, and then outputs a fancy link (in rainbow text using <code>lolcat</code>!!!!)</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/cb00559f1a3f4ce092913f7445bb777c80744eff/config/.local/bin/today"><code>today</code></a> is pretty much just a turbo lazy wrapper around <code>date</code>, because i can never remember the flags for that, nor can i be bothered to learn. without any arguments, <code>today</code> will output the current date in ISO format (for example: 2026-03-05). if you supply the <code>time</code> argument, it includes the current hour &amp; minute in 12 hour time, with an underscore between it and the date. i use this most often in scripting (as you'll see in the next script).</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/1ad38a575b6f39d3ec8ba774d00ee1641d07ded0/config/.local/bin/updiscord"><code>updiscord</code></a> is so incredibly lazy and useful at the same time. since discord distributes <code>.deb</code> files through their web server instead of using a package manager like normal people (to be fair the APT repositories take forever to update, but <em>still!</em> this is a pain!!!!), i automated the download and installation process for getting the new file and installing it with <code>dpkg</code>.</p>
<h3>multimedia (videos, images, audio, etc.)</h3>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/29b6bb9c29db58baf14b2947343a9a2b832cf4f6/config/.local/bin/audiotools"><code>audiotools</code> </a> collects several options for audio processing: <code>flac2mp3</code>, which will convert all FLAC files to the highest quality MP3s possible, and <code>wav2mp3</code>, which does the same but for WAV files.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/98b9686b08968a9059c708dd414b523d1e27cd49/config/.local/bin/switchbluez"><code>switchbluez</code></a> lets me switch between bluetooth audio profiles for my headset. the LXQt GUI for this is really annoying and often requires a few disconnect-and-reconnects of the headset for the right profile to show, so i finally made a script for it after getting really annoyed with the GUI.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/29b6bb9c29db58baf14b2947343a9a2b832cf4f6/config/.local/bin/cbtools"><code>cbtools</code></a> is for managing comic books: <code>cbr2cbz</code> will unzip CBR files into folders, and <code>dir2cbz</code> will zip those resulting folders into <code>CBZ</code> files.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/11462a65786de436d0fe0c1ecc4acd0272854959/config/.local/bin/snsclip"><code>snsclip</code></a> allows for me to spread the good word of <a href="https://girlonthemoon.xyz/ruru/index.html">dambara ruru</a> as fast as possible, by quickly ripping videos from youtube and other platforms, and keeping the resolution under 1080p for a smaller file size.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/4c1fe75d4fd5732effb71be90d20c6df8a89293e/config/.local/bin/vidtools"><code>vidtools</code></a> collects tools related to video processing: <code>avi2mkv</code>, which converts all AVI files to MKV; <code>any2hevc</code>, which converts all MKV files in the current working directory to HEVC encoding; <code>vid2frames</code>, which splits any video in the current working directory into every single frame it consists of (i use this for making GIFs, if the use case is unclear); and <code>vidsubs</code>, which leverages <a href="https://mkvtoolnix.download/"><code>mkvmerge</code></a> to remux MKV files with subtitles of identical file names in the same directory, setting the default subtitle language to English.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/790a91c1ec9818bb0e93b9e1d2587d71854c1035/config/.local/bin/webcamimg"><code>webcamimg</code></a> is exclusively for my laptop. it allows me to take two photos with my webcam on demand. it first turns on my webcam for a few seconds, because if it doesn't, the webcam will immediately turn on when taking the photo, and as it turns on it adjusts to the lighting of the room, which results in extremely dark, barely visible photos. turning it on beforehand momentarily seems to fix that. the photos are taken with <code>ffmpeg</code>.</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/c340b695122e30786aa16f508622f558318dff35/config/.local/bin/ytbest"><code>ytbest</code></a> will download a video from youtube (or any other video platform supported by <code>yt-dlp</code>) in the best audio &amp; video formats possible, while excluding AI upscales (see <a href="https://alexwlchan.net/notes/2025/ignore-ai-scaled-videos/">this post by alexwlchan</a> for info on how to do that).</p>
<p><a href="https://tangled.org/katproto.girlonthemoon.xyz/dotfiles/blob/11462a65786de436d0fe0c1ecc4acd0272854959/config/.local/bin/ytmp3"><code>ytmp3</code></a> is a lazy way for me to get my idol performance MP3s :P in other words, it will extract audio from a video in the best quality possible, outputting it with the video title and an MP3 file extension.</p>
