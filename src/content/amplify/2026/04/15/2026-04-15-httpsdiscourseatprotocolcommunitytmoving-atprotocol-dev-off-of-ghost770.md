<p>I was initially going to not use this category for “software stuff”, but ultimately things like the <code>atprotocol.dev</code> that we used for the 2025 conference, kicked off Tech Talks with, and ultimately used to grow a decent subscriber list, is in fact also community infrastructure.</p>
<h2>AtmosphereConf Website</h2>
<p>This was a great experience in <a href="https://discourse.atprotocol.community/u/essentialrandom.bsky.social">@essentialrandom.bsky.social</a>’s work in getting a great base set, going wild with lots of extra features, and then the community contributing on top.</p>
<p>I had initially been thinking that we could put upcoming events, maybe to support regional events, and basically EVERYTHING here… but realistically, it is a conference website, and we can put <s>wild amounts of</s> effort into it every year and have archives. But the domain is a conference domain, not a broader place to send people to.</p>
<h2>atprotocol dev vs community</h2>
<p>There is a place and a need for developer content, and the <code>atprotocol.dev</code> domain is perfect for this! TODO: see who wants to run with this, in collecting developer resources and topics, as a landing page for getting started, and so on.</p>
<p>I know dev tools is close to <a href="https://discourse.atprotocol.community/u/ngerakines.me">@ngerakines.me</a> heart, <a href="https://discourse.atprotocol.community/u/zzstoatzz.io">@zzstoatzz.io</a> has been hosting dev chats on Sundays, <a href="https://discourse.atprotocol.community/u/trezy.codes">@trezy.codes</a> has a number of tools, <a href="https://discourse.atprotocol.community/u/essentialrandom.bsky.social">@essentialrandom.bsky.social</a> is doing this for her community, and it’s in <a href="https://discourse.atprotocol.community/u/brittanyellich.com">@brittanyellich.com</a>’s dayjob wheelhouse.</p>
<p>I’m a good “I know a guy who knows stuff”, and can MC / host tech talks to a certain degree, but me leading dev tooling when I’m not <s>agent whispering</s> code typing isn’t maybe the best person here.</p>
<p>Which brings us to…</p>
<h2>community landing page</h2>
<p>Maybe we swap out <code>atprotocol.dev</code> for <code>atprotocol.community</code> to make it a broad space for community energy and infrastructure.</p>
<p>The blog at <code>atprotocol.dev</code> is on Ghost today and has a decent subscriber count. There are another 1000 people we’re going to email via Tito as a wrap up post – where do we send them to “subscribe by email”? We’ll make a form where people can stick their email and capture some preferences, and then we can start sending email from…</p>
<p>…a cool new blog aggregator community website.</p>
<p>Here’s my pitch of features:</p>
<ul> <li><strong>atprotocol community blog:</strong> it’s an aggregator, we do some sort of allow list or otherwise figure out which standard.site compatible posts get aggregated there (and maybe it’s actually open, but we only “highlight” certain ones?)</li> <li><strong>mailing list:</strong> we do do email, and “just” email only sign up if people want (conference, events, and atproto community announcements are my rough three categories of lists to optin / out)</li> <li><strong>upcoming events:</strong> we figure out a standard for shared / group calendars and show a calendar building on all the amazing stuff that happened for talks for the conference → manual wiki version is here <a href="https://discourse.atprotocol.community/t/2026-events-atproto-open-social-dev-and-more/293">2026 Events - ATProto, Open Social, Dev and More</a> + we include all the regional events so there is one stop shop of all atproto related events</li> <li><strong>community badges:</strong> much like the badges that <a href="https://discourse.atprotocol.community/u/essentialrandom.bsky.social">@essentialrandom.bsky.social</a> already started, we make some community badges – our friends at npmx want this, others might too, this becomes a pattern / example</li> </ul>
<p>Is this something you want to work on / discuss further? Thoughts on which domains go where? More of a comment than a question?</p>
