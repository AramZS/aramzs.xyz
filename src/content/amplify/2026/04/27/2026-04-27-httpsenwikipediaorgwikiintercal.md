<p>The <b>Compiler Language With No Pronounceable Acronym</b> (<b>INTERCAL</b>) is an <a href="https://en.wikipedia.org/wiki/Esoteric_programming_language">esoteric programming language</a> that was created as a <a href="https://en.wikipedia.org/wiki/Parody">parody</a> by <a href="https://en.wikipedia.org/wiki/Don_Woods_(programmer)">Don Woods</a> and <a href="https://en.wikipedia.org/w/index.php?title=James_M._Lyon&amp;action=edit&amp;redlink=1">James M. Lyon</a>, two <a href="https://en.wikipedia.org/wiki/Princeton_University">Princeton University</a> students, in 1972. It satirizes aspects of the various programming languages at the time,<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-1">[1]</a></sup> as well as the proliferation of proposed language constructs and notations in the 1960s.</p>
<p>There are two maintained implementations of INTERCAL dialects: C-INTERCAL (created in 1990), maintained by <a href="https://en.wikipedia.org/wiki/Eric_S._Raymond">Eric S. Raymond</a><sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-esr-2">[2]</a></sup> and Alex Smith,<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-3">[3]</a></sup> and CLC-INTERCAL, maintained by Claudio Calvelli.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-clc-4">[4]</a></sup> As of March 2021, both implementations were available in the <a href="https://en.wikipedia.org/wiki/Debian">Debian</a> Software Archive.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-5">[5]</a></sup></p>
<h2>History</h2>
<p>According to the original manual by the authors,<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup></p>
<blockquote><p>The full name of the compiler is "Compiler Language With No Pronounceable Acronym", which is, for obvious reasons, abbreviated "INTERCAL".</p></blockquote>
<p>The original Princeton implementation used <a href="https://en.wikipedia.org/wiki/Punched_card">punched cards</a> and the <a href="https://en.wikipedia.org/wiki/EBCDIC">EBCDIC</a> character set. To allow INTERCAL to run on computers using <a href="https://en.wikipedia.org/wiki/ASCII">ASCII</a>, substitutions for two characters had to be made: <code>$</code> substituted for <code>¢</code> as the <i>mingle</i> operator, "represent[ing] the increasing cost of software in relation to hardware", and <code>?</code> was substituted for <code>⊻</code> as the unary <a href="https://en.wikipedia.org/wiki/Exclusive-or">exclusive-or</a> operator to "correctly express the average person's reaction on first encountering exclusive-or".<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup> In recent versions of C-INTERCAL, the older operators are supported as alternatives; INTERCAL programs may now be encoded in <a href="https://en.wikipedia.org/wiki/ASCII">ASCII</a>, <a href="https://en.wikipedia.org/wiki/Latin-1">Latin-1</a>, or <a href="https://en.wikipedia.org/wiki/UTF-8">UTF-8</a>.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-7">[7]</a></sup></p>
<h2>Version numbers</h2>
<p>C-INTERCAL swaps the major and minor version numbers, compared to tradition. The HISTORY<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-8">[8]</a></sup> file shows releases starting at version 0.3 and as of May 2020 having progressed to 0.31, but containing 1.26 between 0.26 and 0.27.</p>
<p>CLC-INTERCAL version numbering scheme was traditional until version 0.06, when it changed to the scheme documented in the <a href="https://en.wikipedia.org/wiki/README">README</a><sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-9">[9]</a></sup> file, which says:</p>
<blockquote> <p> * The term "version" has been replaced by "perversion" for correctness </p><p> * The perversion number consists of a floating-point number with independent signs for the integer and fractional part. Negative fractions indicate pre-escapes (so 1.-94 means "94 pre-escapes to go before 1.00". Or you can just add the numbers together and get 0.06, which is entirely a coincidence since 0.06 is not being developed) </p><p> * The fractional part of a perversion number can be integer or floating point, with a similar meaning for the parts. The current pre-escape is 1.-94.-2 which means "2 pre-pre-escapes to go before pre-escape 1.-94". </p> </blockquote>
<h2>Details</h2>
<p>INTERCAL was intended to be completely different from all other computer languages. Common operations in other languages have cryptic and redundant syntax in INTERCAL. From the INTERCAL Reference Manual:<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup></p>
<blockquote><p>It is a well-known and oft-demonstrated fact that a person whose work is incomprehensible is held in high esteem. For example, if one were to state that the simplest way to store a value of 65536 in a 32-bit INTERCAL variable is: </p><pre>DO :1 &lt;- #0¢#256
</pre><p> any sensible programmer would say that that was absurd. Since this is indeed the simplest method, the programmer would be made to look foolish in front of his boss, who would of course happen to turn up, as bosses are wont to do. The effect would be no less devastating for the programmer having been correct.</p></blockquote>
<p>INTERCAL has many other features designed to make it even more aesthetically unpleasing to the programmer: it uses statements such as "READ OUT", "IGNORE", "FORGET", and modifiers such as "PLEASE". This last keyword provides two reasons for the program's rejection by the compiler: if "PLEASE" does not appear often enough, the program is considered insufficiently polite, and the error message says this; if it appears too often, the program could be rejected as excessively polite. Although this feature existed in the original INTERCAL compiler, it was undocumented.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manualick-10">[10]</a></sup></p>
<p>Despite the language's intentionally obtuse and wordy syntax, INTERCAL is nevertheless <a href="https://en.wikipedia.org/wiki/Turing-complete">Turing-complete</a>: given enough memory, INTERCAL can solve any problem that a <a href="https://en.wikipedia.org/wiki/Universal_Turing_machine">Universal Turing machine</a> can solve. Most implementations of INTERCAL do this very slowly, however. A <a href="https://en.wikipedia.org/wiki/Sieve_of_Eratosthenes">Sieve of Eratosthenes</a> benchmark, computing all prime numbers less than 65536, was tested on a <a href="https://en.wikipedia.org/wiki/Sun_Microsystems">Sun</a> <a href="https://en.wikipedia.org/wiki/SPARCstation_1">SPARCstation 1</a> in 1992. In C, it took less than half a second; the same program in INTERCAL took over seventeen hours.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-stross-11">[11]</a></sup></p>
<h2>Documentation</h2>
<figure><a href="https://en.wikipedia.org/wiki/File:INTERCAL_Circuitous_Diagram.svg"><img alt="" data-file-height="615" data-file-width="466" src="https://upload.wikimedia.org/wikipedia/commons/thumb/4/41/INTERCAL_Circuitous_Diagram.svg/220px-INTERCAL_Circuitous_Diagram.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/4/41/INTERCAL_Circuitous_Diagram.svg/330px-INTERCAL_Circuitous_Diagram.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/4/41/INTERCAL_Circuitous_Diagram.svg/440px-INTERCAL_Circuitous_Diagram.svg.png 2x"/></a></figure>
<p>The "circuitous diagram" from the INTERCAL Reference Manual, purportedly to explain the operation of the "select" operator</p>
<p>The INTERCAL Reference Manual contains many paradoxical, nonsensical, or otherwise humorous instructions:</p>
<blockquote><p> Caution! Under no circumstances confuse the mesh with the interleave operator, except under confusing circumstances! </p></blockquote>
<p>The manual also contains a "<a href="https://en.wikipedia.org/wiki/Tonsil">tonsil</a>", as explained in this footnote: "4) Since all other reference manuals have appendices, it was decided that the INTERCAL manual should contain some other type of removable organ."<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup></p>
<p>The INTERCAL manual gives unusual names to all non-alphanumeric <a href="https://en.wikipedia.org/wiki/ASCII">ASCII</a> characters: single and double quotes are "sparks" and "rabbit ears" respectively. (The exception is the <a href="https://en.wikipedia.org/wiki/Ampersand">ampersand</a>: as the <a href="https://en.wikipedia.org/wiki/Jargon_File">Jargon File</a> states, "what could be sillier?") The assignment operator, represented as an equals sign (INTERCAL's "half mesh") in many other programming languages, is in INTERCAL a left-arrow, <code>&lt;-</code>, made up of an "angle" and a "worm", obviously read as "gets".</p>
<h2>Syntax</h2>
<p>Input (using the <code>WRITE IN</code> instruction) and output (using the <code>READ OUT</code> instruction) do not use the usual formats; in INTERCAL-72, WRITE IN inputs a number written out as digits in English (such as SIX FIVE FIVE THREE FIVE), and READ OUT outputs it in "butchered" <a href="https://en.wikipedia.org/wiki/Roman_numerals">Roman numerals</a>.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup> More recent versions have their own I/O systems.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-clc-4">[4]</a></sup><sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manualick-10">[10]</a></sup></p>
<p>Comments can be achieved by using the inverted statement identifiers involving NOT or N'T; these cause lines to be initially ABSTAINed so that they have no effect.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup> (A line can be ABSTAINed from even if it doesn't have valid syntax; syntax errors happen at runtime, and only then when the line is un-ABSTAINed.)<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup></p>
<h3>Data structures</h3>
<p>INTERCAL-72 (the original version of INTERCAL) had only four <a href="https://en.wikipedia.org/wiki/Data_type">data types</a>: the 16-<a href="https://en.wikipedia.org/wiki/Bit">bit</a> <a href="https://en.wikipedia.org/wiki/Integer">integer</a> (represented with a <code>.</code>, called a "spot"), the 32-bit integer (<code>:</code>, a "twospot"), the <a href="https://en.wikipedia.org/wiki/Array_data_type">array</a> of 16-bit integers (<code>,</code>, a "tail"), and the array of 32-bit integers (<code>;</code>, a "hybrid"). There are 65535 available variables of each type, numbered from <code>.1</code> to <code>.65535</code> for 16-bit integers, for instance. However, each of these variables has its own <a href="https://en.wikipedia.org/wiki/Stack_(data_structure)">stack</a> on which it can be pushed and popped (STASHed and RETRIEVEd, in INTERCAL terminology), increasing the possible complexity of data structures.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup> More modern versions of INTERCAL have by and large kept the same data structures, with appropriate modifications; TriINTERCAL, which modifies the <a href="https://en.wikipedia.org/wiki/Radix">radix</a> with which numbers are represented, can use a 10-<a href="https://en.wikipedia.org/wiki/Trit_(computing)">trit</a> type rather than a 16-bit type,<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manualick-10">[10]</a></sup> and CLC-INTERCAL implements many of its own data structures, such as "classes and lectures", by making the basic data types store more information rather than adding new types.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-clc-4">[4]</a></sup> Arrays are dimensioned by assigning to them as if they were a <a href="https://en.wikipedia.org/wiki/Scalar_(computing)">scalar</a> variable. Constants can also be used, and are represented by a <code>#</code> ("mesh") followed by the constant itself, written as a <a href="https://en.wikipedia.org/wiki/Decimal">decimal</a> number; only integer constants from 0 to 65535 are supported.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup></p>
<h3>Operators</h3>
<p>There are only five operators in INTERCAL-72. Implementations vary in which characters represent which operation, and many accept more than one character, so more than one possibility is given for many of the operators.</p>
<table> INTERCAL operators<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-clc-4">[4]</a></sup><sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup><sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manualick-10">[10]</a></sup> <tbody><tr> <th>Operator </th> <th>INTERCAL-72 characters </th> <th>C-INTERCAL characters </th> <th>CLC-INTERCAL characters </th></tr> <tr> <td>INTERLEAVE / MINGLE </td> <td><code>c</code> <a href="https://en.wikipedia.org/wiki/Backspace">backspace</a> <code>/</code> </td> <td><code>¢</code>, <code>$</code>, <code>c</code> backspace <code>/</code> </td> <td><code>¢</code> </td></tr> <tr> <td>SELECT</td> <td><code>~</code></td> <td><code>~</code></td> <td><code>~</code> </td></tr> <tr> <td>AND</td> <td><code>&amp;</code></td> <td><code>&amp;</code></td> <td><code>&amp;</code> </td></tr> <tr> <td>OR</td> <td><code>V</code></td> <td><code>V</code></td> <td><code>V</code> </td></tr> <tr> <td>XOR </td> <td><code>V</code> backspace <code>-</code> </td> <td><code>V</code> backspace <code>-</code>, <code>?</code>, <code>∀</code> </td> <td><code>V</code> backspace <code>-</code>, <code>¥</code> </td></tr></tbody></table>
<p>Contrary to most other languages, AND, OR, and XOR are <a href="https://en.wikipedia.org/wiki/Unary_operation">unary</a> operators, which work on consecutive bits of their argument; the <a href="https://en.wikipedia.org/wiki/Most_significant_bit">most significant bit</a> of the result is the operator applied to the least significant and most significant bits of the input, the second-most-significant bit of the result is the operator applied to the most and second-most significant bits, the third-most-significant bit of the result is the operator applied to the second-most and third-most bits, and so on. The operator is placed between the punctuation mark specifying a variable name or constant and the number that specifies which variable it is, or just inside grouping marks (i.e. one character later than it would be in programming languages like <a href="https://en.wikipedia.org/wiki/C_(programming_language)">C</a>.) SELECT and INTERLEAVE (which is also known as MINGLE) are infix binary operators; SELECT takes the bits of its first operand that correspond to "1" bits of its second operand and removes the bits that correspond to "0" bits, shifting towards the least significant bit and padding with zeroes (so 51 (1<b>1</b>0<b>0</b>1<b>1</b> in binary) SELECT 21 (10101 in binary) is 5 (<b>101</b> in binary)); MINGLE alternates bits from its first and second operands (in such a way that the least significant bit of its second operand is the least significant bit of the result). There is no operator precedence; grouping marks must be used to disambiguate the precedence where it would otherwise be ambiguous (the grouping marks available are <code>'</code> ("spark"), which matches another spark, and <code>"</code> ("rabbit ears"), which matches another rabbit ears; the programmer is responsible for using these in such a way that they make the expression unambiguous).<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup></p>
<h3>Control structures</h3>
<p>INTERCAL statements all start with a "statement identifier"; in INTERCAL-72, this can be <code>DO</code>, <code>PLEASE</code>, or <code>PLEASE DO</code>, all of which mean the same to the program (but using one of these too heavily causes the program to be rejected, an <a href="https://en.wikipedia.org/wiki/Undocumented_feature">undocumented feature</a> in INTERCAL-72 that was mentioned in the C-INTERCAL manual),<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manualick-10">[10]</a></sup> or an inverted form (with <code>NOT</code> or <code>N'T</code> appended to the identifier).<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup> Backtracking INTERCAL, a modern variant, also allows variants using <code>MAYBE</code> (possibly combined with PLEASE or DO) as a statement identifier, which introduces a choice-point.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-backtrack-12">[12]</a></sup> Before the identifier, an optional line number (an integer enclosed in parentheses) can be given; after the identifier, a percent chance of the line executing can be given in the format <code>%50</code>, which defaults to 100%.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup></p>
<p>In INTERCAL-72, the main control structures are NEXT, RESUME, and FORGET. <code>DO (<i>line</i>) NEXT</code> branches to the line specified, remembering the next line that would be executed if it weren't for the NEXT on a call stack (other identifiers than DO can be used on any statement, DO is given as an example); <code>DO FORGET <i>expression</i></code> removes <i>expression</i> entries from the top of the call stack (this is useful to avoid the error that otherwise happens when there are more than 80 entries), and <code>DO RESUME <i>expression</i></code> removes <i>expression</i> entries from the call stack and jumps to the last line remembered.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup></p>
<p>C-INTERCAL also provides the <a href="https://en.wikipedia.org/wiki/COME_FROM">COME FROM</a> instruction, written <code>DO COME FROM (<i>line</i>)</code>; CLC-INTERCAL and the most recent C-INTERCAL versions also provide computed COME FROM (<code>DO COME FROM <i>expression</i>)</code> and NEXT FROM, which is like COME FROM but also saves a return address on the NEXT STACK.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-clc-4">[4]</a></sup></p>
<p>Alternative ways to affect program flow, originally available in INTERCAL-72, are to use the IGNORE and REMEMBER instructions on variables (which cause writes to the variable to be silently ignored and to take effect again, so that instructions can be disabled by causing them to have no effect), and the ABSTAIN and REINSTATE instructions on lines or on types of statement, causing the lines to have no effect or to have an effect again respectively.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manual72-6">[6]</a></sup></p>
<h3>Hello, world</h3>
<p>The traditional <a href="https://en.wikipedia.org/wiki/Hello_world_program">"Hello, world!" program</a> demonstrates how different INTERCAL is from standard programming languages. In <a href="https://en.wikipedia.org/wiki/C_(programming_language)">C</a>, it could read as follows:</p>
<pre>#include &lt;stdio.h&gt;

int main(void) {
 printf("Hello, world!\n");

 return 0;
}
</pre>
<p>The equivalent program in C-INTERCAL is longer and harder to read:</p>
<pre>DO ,1 &lt;- #13
PLEASE DO ,1 SUB #1 &lt;- #238
DO ,1 SUB #2 &lt;- #108
DO ,1 SUB #3 &lt;- #112
DO ,1 SUB #4 &lt;- #0
DO ,1 SUB #5 &lt;- #64
DO ,1 SUB #6 &lt;- #194
DO ,1 SUB #7 &lt;- #48
PLEASE DO ,1 SUB #8 &lt;- #22
DO ,1 SUB #9 &lt;- #248
DO ,1 SUB #10 &lt;- #168
DO ,1 SUB #11 &lt;- #24
DO ,1 SUB #12 &lt;- #16
DO ,1 SUB #13 &lt;- #162
PLEASE READ OUT ,1
PLEASE GIVE UP
</pre>
<h2>Dialects</h2>
<p>The original Woods–Lyon INTERCAL was very limited in its <a href="https://en.wikipedia.org/wiki/Input/output">input/output</a> capabilities: the only acceptable input were numbers with the digits spelled out, and the only output was an extended version of <a href="https://en.wikipedia.org/wiki/Roman_numerals">Roman numerals</a>.</p>
<p>The C-INTERCAL reimplementation, being available on the Internet, has made the language more popular with devotees of esoteric programming languages.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-nickm-13">[13]</a></sup> The C-INTERCAL dialect has a few differences from original INTERCAL and introduced a few new features, such as a <a href="https://en.wikipedia.org/wiki/COMEFROM">COME FROM</a> statement and a means of doing text I/O based on the Turing Text Model.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manualick-10">[10]</a></sup></p>
<p>The authors of C-INTERCAL also created the TriINTERCAL variant, based on the <a href="https://en.wikipedia.org/wiki/Ternary_numeral_system">Ternary numeral system</a> and generalizing INTERCAL's set of operators.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manualick-10">[10]</a></sup></p>
<p>A more recent variant is Threaded Intercal, which extends the functionality of COME FROM to support <a href="https://en.wikipedia.org/wiki/Thread_(computer_science)">multithreading</a>.<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-manualthick-14">[14]</a></sup></p>
<p>CLC-INTERCAL has a library called INTERNET for networking functionality including being an INTERCAL server, and also includes features such as Quantum Intercal, which enables multi-value calculations in a way purportedly ready for the first <a href="https://en.wikipedia.org/wiki/Quantum_computing"> quantum computers</a>.</p>
<p>In early 2017 a <a href="https://en.wikipedia.org/wiki/.NET">.NET</a> Implementation targeting the <a href="https://en.wikipedia.org/wiki/.NET_Framework">.NET Framework</a> appeared on <a href="https://en.wikipedia.org/wiki/GitHub">GitHub</a>. This implementation supports the creation of standalone binary libraries and interop with other programming languages. <sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-componentintercal-15">[15]</a></sup></p>
<h2>Impact and discussion</h2>
<p>In the article "A Box, Darkly: Obfuscation, Weird Languages, and Code Aesthetics",<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-nickm-13">[13]</a></sup> INTERCAL is described under the heading "Abandon all sanity, ye who enter here: INTERCAL". The compiler and commenting strategy are among the "weird" features described:</p>
<blockquote><p> The compiler, appropriately named "ick", continues the parody. Anything the compiler can't understand, which in a normal language would result in a compilation error, is just skipped. This "forgiving" feature makes finding bugs very difficult; it also introduces a unique system for adding program comments. The programmer merely inserts non-compileable text anywhere in the program, being careful not to accidentally embed a bit of valid code in the middle of their comment.</p></blockquote>
<p>In "Technomasochism",<sup><a href="https://en.wikipedia.org/wiki/INTERCAL#cite_note-levb-16">[16]</a></sup> Lev Bratishenko characterizes the INTERCAL compiler as a dominatrix:</p>
<blockquote><p>If PLEASE was not encountered often enough, the program would be rejected; that is, ignored without explanation by the compiler. Too often and it would still be rejected, this time for sniveling. Combined with other words that are rarely used in programming languages but appear as statements in INTERCAL, the code reads like someone pleading.</p></blockquote>
<h2>Popular culture</h2>
<p>The Nitrome Enjoyment System, a fictional video game console created by British indie game developer <a href="https://en.wikipedia.org/wiki/Nitrome">Nitrome</a>, has games which are programmed in INTERCAL.</p>
