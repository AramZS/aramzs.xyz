<p>littlefoot is a lightweight JavaScript library that creates exceptional footnotes.</p>
<figure><img alt="Screen recording of littlefoot in action" src="https://littlefoot.js.org/assets/littlefoot-demo.gif"/></figure>
<p>Simply include the code on your pages and footnotes will be detected automatically and improved in the following ways:</p>
<ul> <li> <p>Links to footnotes will be replaced with clickable/tappable buttons, making them substantially easier to hit.</p> </li> <li> <p>Footnote content will appear in a popover directly beside the footnote button when it is clicked/tapped, which cuts out the annoying bouncing around the page that footnotes typically result in.</p> </li> <li> <p>The active popovers will be resized and repositioned to ensure that they continue to be completely visible on-screen and aesthetically pleasing: this makes it perfect for mobile devices and responsive designs.</p> </li> <li> <p>Supports the markup generated by MultiMarkdown by default.</p> </li> </ul>
<p>This project includes the script itself and a default style to apply to the footnote buttons and popovers that are eventually generated. There are also a variety of additional styles that illustrate some of the possibilities for styling these components.</p>
<p>littlefoot was forked from <a href="https://github.com/lemonmade/bigfoot/">Bigfoot.js</a> by <a href="https://github.com/lemonmade">Chris Sauvé</a>. Unlike Bigfoot.js, littlefoot does not require jQuery.</p>
<h2>Installation</h2>
<p>Install using NPM:</p>
<pre><code>npm install --save littlefoot
</code></pre>
<p>Install using Yarn:</p>
<pre><code>yarn add littlefoot
</code></pre>
<h2>Usage</h2>
<p>The script will work with a wide array of markup, but you will need to make sure that your footnote content/link markup at least resembles the markup shown below, which is the format generated by MultiMarkdown:</p>
<pre><code>&lt;!-- Links --&gt;
&lt;p&gt;
  &lt;sup id="fnref:1"&gt;
    &lt;a href="#fn:1"&gt;1&lt;/a&gt;
  &lt;/sup&gt;
&lt;/p&gt;

&lt;!-- Footnote List --&gt;
&lt;div class="footnotes"&gt;
  &lt;ol&gt;
    &lt;li class="footnote" id="fn:1"&gt;
      &lt;p&gt;footnote. &lt;a href="#fnref:1" title="return to article"&gt; ↩&lt;/a&gt;&lt;/p&gt;
      &lt;p&gt;&lt;/p&gt;
    &lt;/li&gt;
  &lt;/ol&gt;
&lt;/div&gt;
</code></pre>

<h2>Options</h2>
<p>The script has many configurable options from having popovers instantiated on hover, to allowing multiple active footnotes, to setting specific timeouts for popover activation and dismissal. It also returns an object that allows you to activate and dismiss popovers.</p>
<h3><code>activateCallback</code></h3>
<p>Specifies a function to call on a footnote popover that is being activated (after it is added to the DOM). The function will be passed two arguments: the <code>popover</code> DOM element, and the <code>button</code> that was activated to show the popover. This option can be useful for adding classes or styling information on the popover.</p>
<p>Default: <code>undefined</code></p>
<h3><code>dismissCallback</code></h3>
<p>Specifies a function to call on a footnote popover that is being dismissed (just before it is removed from the DOM). The function will be passed two arguments: the <code>popover</code> DOM element, and the related <code>button</code>. This option can be useful for removing classes on the popover.</p>
<p>Default: <code>undefined</code></p>
<h3><code>activateDelay</code></h3>
<p>Sets a delay between the activation of the footnote button and the activation of the actual footnote content.</p>
<p>Default: <code>100</code></p>
<h3><code>activateOnHover</code></h3>
<p>Specifies whether or not the footnote content will be activated when the associated button is hovered over.</p>
<p>Default: <code>false</code></p>
<h3><code>allowDuplicates</code></h3>
<p>Determines whether or not a footnote can be used as the content for multiple footnote buttons. Many content management systems will, on a blog’s main page, load every article chronologically without any adjustments to the article markup. This can cause issues if multiple footnotes have the same ID: the footnote content is identified by the fragment identifier in the <code>href</code> attribute of the footnote link, so multiple identical IDs can result in the same footnote content being used for different footnote links. This option prevents this by using a footnote as the content for at most one footnote button.</p>
<p>Default: <code>false</code></p>
<h3><code>allowMultiple</code></h3>
<p>Specifies whether or not multiple footnote popovers can be active simultaneously.</p>
<p>Default: <code>false</code></p>
<h3><code>anchorPattern</code></h3>
<p>Specifies the pattern that must be matched by the anchor element’s <code>href</code> attribute for it to be considered a footnote link. This is used in filtering all links down to just those with a footnote.</p>
<p>Default: <code>/(fn|footnote|note)[:\-_\d]/gi</code></p>
<h3><code>anchorParentSelector</code></h3>
<p>The selector for the parent of the footnote link. This is really only necessary when you want to get rid of that element — for instance, when the link is inside a <code>sup</code> tag. This tag and the link itself will be joined together for attribute from which you can drawn in your markup for footnotes/buttons.</p>
<p>Default: <code>sup</code></p>
<h3><code>dismissDelay</code></h3>
<p>When the footnote content is being removed this option specifies how long after the active class is removed from the footnote before the element is actually removed from the DOM.</p>
<p>Default: <code>500</code></p>
<h3><code>dismissOnUnhover</code></h3>
<p>Determines whether footnotes that were presented when hovering on a footnote button are removed once the footnote button or footnote popover is un-hovered.</p>
<p>Default: <code>false</code></p>
<h3><code>footnoteSelector</code></h3>
<p>The element that contains the footnote content. This element will be hidden and given a <code>littlefoot--print</code> class once littlefoot has finished with it.</p>
<p>Default: <code>'li'</code></p>
<h3><code>hoverDelay</code></h3>
<p>If <code>dismissOnUnhover</code> is true, this specifies the amount of time (in milliseconds) that must pass after the footnote button/content is un-hovered before the footnote is removed.</p>
<p>Default: <code>250</code></p>
<h3><code>numberResetSelector</code></h3>
<p>A string representing the selector at which you would like the numbering of footnotes to restart at 1. For example, you may be using the numbered style of footnote and wish to have the numbers restart for each <code>&lt;article&gt;</code> on your main page with a class of <code>'article-container'</code> In this case, you would set this option to <code>'article.article-container'</code> (or an equivalent CSS selector). Leaving the option undefined will simply number all footnotes on a given page sequentially.</p>
<p>Default: <code>undefined</code></p>
<h3><code>scope</code></h3>
<p>If any truthy value is provided, only the footnotes within the scope you define will be affected by the script. The scope should be a selector string, as you would typically use in jQuery. For example, setting a scope of <code>'.littlefoot-active'</code> would work only on those elements with an ancestor that has a class of <code>littlefoot-active</code>.</p>
<p>Default: <code>undefined</code></p>
<h3><code>contentTemplate</code></h3>
<p>A template for the markup of the footnote content popovers. It’s best not to change this too much; the library relies on the class names and hierarchy of the default markup to do its work. However, you can add information to the rendered markup by adding string literals or one or more of the following variables:</p>
<ul> <li><code>content</code>: The HTML markup of the original footnote.</li> <li><code>number</code>: The footnote number (sequential ordering of all footnotes within an element matching the <code>numberResetSelector</code> option).</li> <li><code>id</code>: The footnote identifier (sequential ordering of all footnotes on the page, starting from 1).</li> </ul>
<p>Variables should be added between interpolation delimiters. For example, <code>&lt;% content %&gt;</code>.</p>
<p>Default:</p>
<h3><code>buttonTemplate</code></h3>
<p>A template for the markup of the footnote button. Again, try not to remove any elements from the markup, but add as much as you like.</p>
<ul> <li><code>content</code>: The escaped HTML markup of the original footnote.</li> <li><code>id</code>: The footnote identifier (sequential ordering of all footnotes on the page, starting from 1).</li> <li><code>number</code>: The footnote number (sequential ordering of all footnotes within an element matching the <code>numberResetSelector</code> option).</li> <li><code>reference</code>: The footnote reference used to populate the ID attribute.</li> </ul>
<p>Variables should be added between interpolation delimiters. For example, <code>&lt;% content %&gt;</code>.</p>
<p>Default:</p>

<h2>Methods</h2>
<p>Running the function will return an object that can be stored and used to manipulate the footnote buttons/content. The following methods are available in this return object:</p>
<h3><code>dismiss([footnoteId[, timeout]])</code></h3>
<p>This function will close the popover matching the footnote ID. If omitted, all popovers are dismissed. <code>timeout</code> specifies the amount of time after the footnote’s active class is removed before the element itself is removed. If excluded, <code>timeout</code> will default to the <code>dismissDelay</code> option.</p>
<h3><code>activate(footnoteId[, timeout])</code></h3>
<p>This will activate the footnote button (and its associated popover) matching the footnote ID. <code>timeout</code> specifies the amount of time before the footnote’s active class is applied. If excluded, <code>timeout</code> will default to the <code>activateDelay</code> option.</p>
<h3><code>getSetting(key)</code></h3>
<p>Returns the instance setting matching the provided string key.</p>
<h3><code>updateSetting(key, value)</code></h3>
<p>Updates the littlefoot instance settings for the provided string key with a new value.</p>
<h3><code>unmount()</code></h3>
<p>This will disable littlefoot and restore the document to its original structure, clearing event handlers. Once <code>unmount()</code> is called, all other methods provided in the littlefoot instance will stop working, requiring you to call the <code>littlefoot()</code> function again.</p>
